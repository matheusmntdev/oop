Orientação a Objetos com PHP

A Orientação a Objetos (OO) é um paradigma de programação que organiza o código em "objetos". Um objeto é uma instância de uma classe, que pode conter atributos e métodos. O PHP suporta a programação orientada a objetos desde sua versão 5. Vamos entender como usar esses conceitos em PHP de forma prática.

1. Conceitos Fundamentais

1.1 Classe

Uma classe é o molde ou estrutura que define um tipo de objeto. Ela contém atributos (também chamados de propriedades) e métodos (também chamados de funções).


class Carro {
    public $modelo;
    public $cor;
    
    public function ligar() {
        echo "O carro está ligado!";
    }
}

1.2 Objeto

Um objeto é uma instância de uma classe. Para criar um objeto, usamos a palavra-chave new.


$meuCarro = new Carro();
$meuCarro->modelo = "Fusca";
$meuCarro->cor = "Azul";
$meuCarro->ligar();

1.3 Atributos (Propriedades)

Atributos são as variáveis associadas a um objeto. Eles podem ter diferentes níveis de acesso:

  • public: Atributo acessível de qualquer lugar.
  • protected: Atributo acessível dentro da classe e suas subclasses.
  • private: Atributo acessível apenas dentro da classe.

1.4 Métodos

Métodos são funções que pertencem a uma classe. Eles podem ser chamados por um objeto e podem modificar os atributos desse objeto.


class Pessoa {
    public $nome;
    
    public function dizerOla() {
        echo "Olá, " . $this->nome;
    }
}

$pessoa = new Pessoa();
$pessoa->nome = "João";
$pessoa->dizerOla();

2. Encapsulamento

O encapsulamento é um dos pilares da programação orientada a objetos. Ele diz respeito à forma como os dados são protegidos e acessados. Podemos controlar o acesso aos atributos de uma classe utilizando métodos especiais como getters e setters.


class Produto {
    private $preco;
    
    public function setPreco($preco) {
        $this->preco = $preco;
    }
    
    public function getPreco() {
        return $this->preco;
    }
}

$produto = new Produto();
$produto->setPreco(100);
echo $produto->getPreco();

3. Herança

Herança permite que uma classe herde propriedades e métodos de outra. Isso facilita a reutilização de código e cria uma hierarquia entre as classes.


class Animal {
    public $nome;
    
    public function fazerSom() {
        echo "Som do animal";
    }
}

class Cachorro extends Animal {
    public function fazerSom() {
        echo "Au Au!";
    }
}

$cachorro = new Cachorro();
$cachorro->fazerSom();  // Saída: Au Au!

4. Polimorfismo

O polimorfismo permite que um objeto de uma classe derivada seja tratado como se fosse da classe base. Ele pode alterar o comportamento de um método da classe base.


class Veiculo {
    public function andar() {
        echo "O veículo está andando!";
    }
}

class Carro extends Veiculo {
    public function andar() {
        echo "O carro está andando!";
    }
}

$veiculo = new Veiculo();
$carro = new Carro();

$veiculo->andar();  // Saída: O veículo está andando!
$carro->andar();    // Saída: O carro está andando!

5. Abstração

A abstração permite esconder os detalhes da implementação e mostrar apenas o que é necessário. Usamos classes abstratas e interfaces para definir comportamentos comuns, mas sem implementação detalhada.

5.1 Classe Abstrata

Uma classe abstrata não pode ser instanciada diretamente e pode ter métodos abstratos que devem ser implementados pelas classes filhas.


abstract class Forma {
    abstract public function calcularArea();
}

class Quadrado extends Forma {
    private $lado;
    
    public function __construct($lado) {
        $this->lado = $lado;
    }
    
    public function calcularArea() {
        return $this->lado * $this->lado;
    }
}

$quadrado = new Quadrado(5);
echo $quadrado->calcularArea();  // Saída: 25

5.2 Interface

Uma interface define um contrato que deve ser seguido pelas classes que a implementam, garantindo que certos métodos sejam implementados.


interface Pagamento {
    public function pagar();
}

class Cartao implements Pagamento {
    public function pagar() {
        echo "Pagamento realizado com cartão.";
    }
}

$pagamento = new Cartao();
$pagamento->pagar();  // Saída: Pagamento realizado com cartão.

6. Construtores e Destrutores

O construtor é um método especial chamado automaticamente quando um objeto é instanciado. O destruidor é chamado quando o objeto é destruído.


class Livro {
    public $titulo;
    
    public function __construct($titulo) {
        $this->titulo = $titulo;
    }
    
    public function __destruct() {
        echo "O livro " . $this->titulo . " foi destruído.";
    }
}

$livro = new Livro("PHP para Iniciantes");
// Saída: O livro PHP para Iniciantes foi destruído (quando o objeto sair de escopo).

7. Traits

Traits são um mecanismo para reutilizar código em múltiplas classes. Eles não podem ser instanciados, mas podem ser incluídos em várias classes.


trait Log {
    public function logar($mensagem) {
        echo "Log: " . $mensagem;
    }
}

class Usuario {
    use Log;
}

$usuario = new Usuario();
$usuario->logar("Usuário logado.");
